Error prevention

"Even better than good error messages is a careful design which prevents a problem from occurring in the first place. Either eliminate error-prone conditions or check for them and present users with a confirmation option before they commit to the action."

Users are prone to perform accidental actions which may result in entering a error state. Handling errors is time consuming for users and is a frustrating experience. Whenever possible, it is simply better to not allow users to enter errors in the first place. This can be achieved through robust web design. Examples include form validation, confirmation dialogue, and minimalization of irrelvant options.

Forms Errors

Sometimes users select options on forms by mistake. For destructive options such as form clearing, it is a good idea to confirm the user's action. In the prospectus form, there are selectable reset options for filled selectors. When the reset option is selected, all selectors will be cleared without warning. This reset error is easily prevented if the reset option was moved outside of the option selectors. It may also be prevented if a dialogue appeared to confirm the user's action.

Forms Error Prevention

Form validation may be done premptively and can notify the user of errors before the form is submitted. One example would be the email text field. It will attempt to validate its value, checking whether it contains a valid email syntax. If it find the supplied value is invalid, the field is highlighted in a red border.

For forms, certain inputs should only be revealed when relevant. This prevents the users for entering uncessesary information, minimizing the chance of inputing incorrect data. One example of this would be "I would like to receive information by" option on the prospectus form. The form will reveal inputs for a mailing addres only if the user selects the "post" option.

No Dead Links

It is very hard to find a dead link on the site. Their web developers have done a good job keeping the links up to date. We have not found a way to visit the site's 404 page through normal means of navigation.
