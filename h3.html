User control and freedom

"Users often choose system functions by mistake and will need a clearly marked "emergency exit" to leave the unwanted state without having to go through an extended dialogue. Support undo and redo."

Users need to be able to navigate through different areas of the site and should be able to find their way to a certain page or area. Common examples of this heuristic are navbars and a variety of actions like undo, remove, update, back/forward buttons, exit, close, etc.

Navigating through the site

For navigation, the only thing that's consistent is a constant control at the top. There's usually a logo in the top left and then right under thre's a navbar that lets you navigate home, to the indexes, and then to the map and directions. However, we found some bugs with this system. The logo in the search page doesn't lead you back to the original home page, but stays on the search page.

On many pages, there is a fixed left sidebar that lets you jump to other sections. The only problem is that it doesn't show subsections, so you have to click/go to the section to be able to see the subsections. This is a little bit problematic as it doesn't give the user full view of the areas they can go to unless they go into the bigger section.
