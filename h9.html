Help Users Recognize, Diagnose, and Recover from Errors

"Error messages should be expressed in plain language (no codes), precisely indicate the problem, and constructively suggest a solution."

There will be times when users encounter errors on a webpage. The user may have visited a page which does not exists or may have entered incorrect information on a form. When an error happens, the resulting error message should be helpful to the user. The message should be precise about the problem and speak in terms the user understands. Ideally, it needs to be helpful to offer potential solutions for the error.
